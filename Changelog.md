# Changelog

## [0.2.0] - Release Date: 2026-02-15
- **Added issue assignment**: Users can now assign team members to issues when creating them in Taiga.
- **Added tags support**: Issues can now be tagged with project tags for better organization.
- **Improved issue creation interface**: Enhanced UI with image preview, close button, loading transitions, and better visual feedback.
- **UI improvements**: Fixed content button pointer, input text color, and popup width adjustments.

## [0.1.3] - Release Date: 2026-02-04
- **Fixed initial sign in lock**: Resolved an issue where users were unable to sign in for the first time due to a locking screen problem.

## [0.1.2] - Release Date: 2026-01-29
- **Improved session handling**: Enhanced the session management to handle invalid sessions more gracefully, reducing the need for users to reset their settings.

## [0.1.1] - Release Date: 2026-01-28
- **Fixed taiga auth loop**: The auth loop issue has been resolved, but there are still some problems with invalid sessions, forcing the user to reset their settings and start over the setup process. This will be addressed in future updates which requires significant changes to the auth flow and data structures.

## [0.1.0] - Release Date: 2026-01-16
- **Initial release**: First stable version with the Taiga integration.
